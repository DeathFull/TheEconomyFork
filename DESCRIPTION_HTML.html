EconomySystem – Hytale Economy Plugin

EconomySystem is a complete economy plugin for Hytale servers, featuring persistent player balances, money commands, admin tools, reward systems (block breaks + monster kills), a full shop GUI, player-to-player shops, localization support (EN/PT/ES/RU), MySQL database support, and a public API for easy integration with other mods.

‚úÖ Main Features

  • Persistent Balances (JSON/MySQL) saved automatically every 30 seconds
  • MySQL Database Support: Full MySQL integration for scalable server deployments
  • Starter Money: New players receive a configurable starting balance (JSON config)
  • Safe & Stable: Thread-safe balance handling with secure disk saving
  • Shop System: Full server shop with GUI, buy/sell support, and live item loading
  • Player Shop System: Player-to-player shops with GUI, stock management, and persistence
  • Translations: Multi-language support (EN / PT / ES / RU)
  • Advanced Currency Formatting: Custom symbol, 1.000,00 format, and short values (1k, 1kk, 1kkk)
  • Rewards System: Earn money from breaking ores/wood and killing monsters
  • Console Support: Admin commands work from server console
  • Plugin's own scoreboard
  • The plugin has now been adapted to a color-coding system for messages similar to that of Minecraft.

üóÑÔ∏è MySQL Database Support

‚ú® New Feature: EconomySystem now includes full MySQL database support for scalable server deployments!

MySQL Configuration

Configure MySQL settings in EconomySystem.json:

  • EnableMySQL - Enable/disable MySQL storage (default: false)
  • MySQLHost - MySQL server host (default: "localhost")
  • MySQLPort - MySQL server port (default: 3306)
  • MySQLUser - MySQL username (default: "root")
  • MySQLPassword - MySQL password (default: "")
  • MySQLDatabaseName - Database name (default: "theeconomy")
  • MySQLTableName - Table name for player balances (default: "bank")
  • MySQLAdminShopTableName - Table name for admin shop (default: "adminshop")
  • MySQLPlayerShopTableName - Table name for player shops (default: "playershop")

MySQL Features

  • ‚úÖ Full MySQL integration for all data storage
  • ‚úÖ Automatic table creation on first connection
  • ‚úÖ Seamless migration between JSON and MySQL
  • ‚úÖ Thread-safe database operations
  • ‚úÖ Automatic connection pooling and management
  • ‚úÖ Supports all plugin features (balances, admin shop, player shops)
  • ‚úÖ Data persists across server restarts
  • ‚úÖ Ideal for large servers with many players

Storage Modes

  • JSON Mode (Default): Data stored in JSON files (good for small/medium servers)
  • MySQL Mode: Data stored in MySQL database (recommended for large servers)
  • You can switch between modes at any time - the plugin handles migration automatically

üîå Public API (EconomyAPI)

EconomySystem includes a public API located at:
com.economy.api.EconomyAPI

Available API Methods

Balance Queries

  • getBalance(UUID playerUUID)
  • getBalance(String playerName)

Balance Management

  • setBalance(UUID playerUUID, double value)
  • setBalance(String playerName, double value) (returns boolean)
  • addBalance(UUID playerUUID, double value)
  • addBalance(String playerName, double value) (returns boolean)
  • removeBalance(UUID playerUUID, double value) (returns boolean)
  • removeBalance(String playerName, double value) (returns boolean)

Balance Checks

  • hasBalance(UUID playerUUID, double value)
  • hasBalance(String playerName, double value)

Player Info Helpers

  • getPlayerName(UUID playerUUID)
  • getPlayerUUID(String playerName)

Formatted Balance

  • getFormattedBalance(UUID playerUUID)
  • getFormattedBalance(String playerName)

Shop Management

  • openShop(Player player) (returns boolean)
    • Opens the shop for a player
    • Automatically checks if shop is enabled in configuration
    • Returns true if shop was opened successfully, false otherwise

API Example

 
import com.economy.api.EconomyAPI;
import com.hypixel.hytale.server.core.entity.entities.Player;
import java.util.UUID;

// Get API instance
EconomyAPI api = EconomyAPI.getInstance();

// Balance queries
double balance = api.getBalance(playerUUID);
double balanceByName = api.getBalance("PlayerName");

// Balance management
api.setBalance(playerUUID, 1000.0);
api.addBalance(playerUUID, 100.0);
boolean success = api.removeBalance(playerUUID, 50.0);

// Balance checks
boolean hasEnough = api.hasBalance(playerUUID, 200.0);

// Formatted balance
String formatted = api.getFormattedBalance(playerUUID);

// Player info
String name = api.getPlayerName(playerUUID);
UUID uuid = api.getPlayerUUID("PlayerName");

// Open shop (in an event or command)
public void onPlayerCommand(Player player) {
    EconomyAPI api = EconomyAPI.getInstance();
    
    // Check balance
    double balance = api.getBalance(player.getUuid());
    
    // Add money
    api.addBalance(player.getUuid(), 100.0);
    
    // Open administrative shop
    if (api.openShop(player)) {
        // Shop opened successfully
    } else {
        // Failed to open shop (may be disabled or player offline)
    }
}

üîó Placeholder API

EconomySystem includes a public Placeholder API for other plugins:

com.economy.util.PlaceholderAPI

Available Placeholders

  • %balance% - Player balance (numeric value)
  • %balance_formated% - Player balance (formatted with currency symbol)

Placeholder API Methods

  • PlaceholderAPI.getInstance().replacePlaceholders(UUID playerUUID, String text)
  • PlaceholderAPI.getInstance().replacePlaceholders(String playerName, String text)
  • PlaceholderAPI.getInstance().containsPlaceholders(String text)

Placeholder Example

 
PlaceholderAPI placeholderAPI = PlaceholderAPI.getInstance();

String text = "Your balance is %balance_formated%";
String result = placeholderAPI.replacePlaceholders(playerUUID, text);
// Result: "Your balance is $1,234.56"
